# KEV-CVE-Mapper-Ranker (KEVMap)

MSSP/SOC-focused dashboard for **CISA Known Exploited Vulnerabilities (KEV)** with **NVD CVE** enrichment (CVSS scores, severity), **EPSS** exploit-probability scores, breach news, AI-generated daily executive briefings, and ready-to-adapt threat-hunting queries.

Runs as a **single Cloudflare Worker** — the Worker serves the API and the React SPA, stores everything in D1 (Cloudflare's SQLite), and does all background enrichment on Cron Triggers. Fits the Workers free plan.

## Quick start (local dev)

```bash
# Terminal 1: the Worker (API + data layer, port 8787)
cd worker
npm install
npm run db:migrate:local        # one-time: create the local D1 schema
cp .dev.vars.example .dev.vars  # optional: add API keys
npm run dev

# Terminal 2: frontend (port 5173, proxies /api to the Worker)
cd webui && npm install && npm run dev
```

Open http://localhost:5173. The dashboard works immediately; CVSS/EPSS scores fill in as enrichment runs. In local dev, cron ticks don't fire on their own — kick one with:

```bash
curl "http://localhost:8787/__scheduled?cron=*%2F5+*+*+*+*"
```

**API keys** (all optional, all server-side — local dev in `worker/.dev.vars`, production via `wrangler secret put`):

| Key | Needed for | Notes |
|---|---|---|
| `NVD_API_KEY` | Faster CVE enrichment | Free from [NVD](https://nvd.nist.gov/developers/request-an-api-key) |
| `SEARCH_API_KEY` | News/links column | [Serper](https://serper.dev) search API |
| `GROQ_API_KEY` | Daily briefing + hunting queries | [Groq](https://groq.com) free tier works |

## Pages

- **Command Center** (`/`) — at-a-glance stats, trend charts, overdue/due-soon SLA tracking, and a Mastodon "Community Pulse" infosec feed
- **KEV Catalog** (`/catalog`) — searchable, filterable, sortable table of KEV entries joined with CVSS + EPSS data; per-CVE news links; CSV and PDF export; print-friendly report view
- **Breaches** (`/breaches`) — curated feed of recent breach/incident news aggregated from trusted RSS sources (Krebs on Security, The Hacker News, BleepingComputer, SecurityWeek, Dark Reading, and more)
- **Briefing** (`/briefing`) — daily executive intelligence briefing generated by Groq (Llama), grounded in the live KEV catalog and breach headlines, archived per-day with a 30-day gap-free backfill
- **Hunting** (`/hunting`) — the briefing's detection queries (Microsoft Sentinel/Defender KQL, Cortex XDR XQL, Splunk SPL) on their own archived page

## How it works

- **Frontend SPA** (Vite + React + TypeScript) under `webui/` — no client-side API keys, no direct calls to CISA/NVD
- **Cloudflare Worker** under `worker/` — fetches the [CISA KEV catalog](https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json), enriches via [NVD CVE API 2.0](https://nvd.nist.gov/developers/vulnerabilities) in cron-driven batches, adds [EPSS](https://www.first.org/epss/) scores with 30-day history, and persists everything in **D1**
- Rows you're actually looking at jump the enrichment queue (`/api/nvd/prioritize`), so scores appear in seconds even while a full catalog backfill is running

## Deployment

Push to `main` builds and deploys via GitHub Actions (`CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID` secrets), or deploy by hand with `cd webui && npm run build && cd ../worker && npm run deploy`. One-time D1/secrets setup: see [DEPLOY.md](DEPLOY.md).
